# n8n workflows

Workflow definitions for the Aegis platform. n8n itself runs on Modal
(`modal_app/n8n_orchestrator.py`); these JSON files are the importable
workflow exports.

## Stable URLs

| Surface | URL |
|---|---|
| n8n editor + login | `https://transilience--aegis-n8n-server.modal.run` |
| Brand-monitoring webhook | `https://transilience--aegis-n8n-server.modal.run/webhook/run-brand-sweep` |
| DLR webhook | `https://transilience--aegis-n8n-server.modal.run/webhook/run-dlr-sweep` |
| OSINT API (Sherlock/Holehe/etc.) | `https://transilience--aegis-osint-api-web.modal.run` |

## Importing a workflow

1. Log in to n8n at `https://transilience--aegis-n8n-server.modal.run`
2. Top-right menu → **Import from file**
3. Pick `n8n/workflows/brand_monitoring.json` (or `dlr_monitoring.json`)
4. The workflow opens — review nodes, **save**, then **activate** (toggle top-right)
5. Webhook URL becomes live; test from the admin panel or via curl

## Required Modal Secret (`aegis-n8n-env`)

Workflows reference these via `{{ $env.NAME }}`. Create the secret once with
`modal secret create aegis-n8n-env` and these keys:

```
N8N_BASIC_AUTH_USER          # login user
N8N_BASIC_AUTH_PASSWORD      # long random
N8N_ENCRYPTION_KEY           # 32-byte hex — credentials are sealed with this
APIFY_TOKEN                  # Apify basic plan token
HIBP_API_KEY                 # HaveIBeenPwned enterprise key
ANTHROPIC_API_KEY            # Anthropic API key (Claude Haiku 4.5)
SUPABASE_URL                 # https://pccbqjissbuogvfukscv.supabase.co
SUPABASE_SERVICE_ROLE_KEY    # service role JWT
OSINT_API_BASE               # https://transilience--aegis-osint-api-web.modal.run
```

Workflows do **not** use n8n's credential system — every API call uses
`$env` interpolation, so no per-credential setup in the n8n UI is needed.

## Test from curl

After importing + activating `Brand Monitoring`, kick a run:

```bash
RUN_ID=$(uuidgen)
curl -X POST https://transilience--aegis-n8n-server.modal.run/webhook/run-brand-sweep \
  -H 'Content-Type: application/json' \
  -d "$(cat <<EOF
{
  "run_id": "$RUN_ID",
  "tenant_id": "<tenant-uuid-from-supabase>",
  "brand": "CreditAccess Grameen",
  "owned_domains": ["creditaccessgrameen.in"],
  "whitelisted_handles": ["@creditaccessgrameen"],
  "executive_emails": []
}
EOF
)"
```

Watch live progress at `https://transilience--aegis-n8n-server.modal.run/executions`.
Findings land in Supabase `findings` table; HIBP breaches in `dlr_records`.

## Webhook payload shape

### `/webhook/run-brand-sweep`
```json
{
  "run_id": "uuid — pre-generated by the admin panel before triggering",
  "tenant_id": "uuid — must exist in tenants table",
  "brand": "CreditAccess Grameen",
  "owned_domains": ["creditaccessgrameen.in"],
  "whitelisted_handles": ["@creditaccessgrameen"],
  "executive_emails": ["ceo@creditaccessgrameen.in"]
}
```

### `/webhook/run-dlr-sweep`
Same payload shape; `executive_emails` are reserved for a follow-up that
calls `/v3/breachedaccount/{email}` per executive.

## Workflow shapes

### Brand Monitoring
```
Webhook → Mark Running → fan-out
  ├─ Apify google-search-scraper       (phishing / typosquats / scams / Telegram)
  ├─ Apify instagram-search-scraper    (fake handles)
  ├─ Modal Kali /sherlock              (username footprint across 400+ sites)
  └─ HIBP /breaches?domain=...         (credential exposure)
→ Merge → Flatten Findings (JS) → Anthropic Claude Haiku
→ Parse AI Output → Has Findings? → Bulk Insert findings → Mark Completed → Respond
```

### DLR Monitoring
```
Webhook → Mark Running → HIBP /breaches → Normalize Breaches (JS)
→ Has Breaches? → Bulk Insert dlr_records → Mark Completed → Respond
```

## Adding a new service workflow (Phase B+)

Pattern: clone `brand_monitoring.json`, swap the source nodes (Apify actor
URL + body), keep Merge → Flatten → Anthropic → Insert → Complete spine.
Each new service is ~30-60 minutes of n8n canvas work + ~1h of testing.
